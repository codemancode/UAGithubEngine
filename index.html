UAGithubEngine

Coating the Github API in steaming hot Cocoa-ey goodness

What?

UAGithubEngine is a practically-complete* Cocoa wrapper around the Github API. Simply add the engine files to your project and implement a few delegate methods: now you're free to enjoy the ease of accessing the Github API using native objects like NSDictionarys, NSDates and NSStrings, using familiar Cocoa design patterns you know and love.

How?

UAGithubEngine *engine = [[UAGithubEngine alloc] 
			initWithUsername:@"auser" apiKey:@"akey" delegate:self];
			
[engine getUser:@"owainhunt"];
[engine addComment:@"Awesomeness." toIssue:@"owainhunt/UAGithubEngine/1"];

That's the short version, obviously. There's more in the readme.

Why?

Github's awesome. So is Cocoa. You want to combine the two in a magical union of awesomitude.
Seriously though, UAGithubEngine takes the tricky, low-level bumph out of developing Mac and iOS software for Github, letting you focus on what you really want to do...make great apps.

Want.

$ git clone git://github.com/owainhunt/UAGithubEngine

*UAGithubEngine does not implement the network graph or Gist APIs.
